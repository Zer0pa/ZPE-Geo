# Support

<p>
  <img src="../.github/assets/readme/section-bars/where-to-go.svg" alt="WHERE TO GO" width="100%">
</p>

| What you need | Where to go |
| --- | --- |
| Repo-boundary or documentation bug | Open an issue or send a scoped note with exact file paths and symptoms |
| Proof-surface dispute | Include the exact artifact path and the contradicting file |
| Package import or install issue | Include the install command, environment, and failing traceback |
| Security vulnerability or secret leak | `architects@zer0pa.ai` and follow `SECURITY.md` |
| License or legal-boundary question | `architects@zer0pa.ai`; `LICENSE` remains the legal source of truth |

<p>
  <img src="../.github/assets/readme/section-bars/what-gets-a-fast-response.svg" alt="WHAT GETS A FAST RESPONSE" width="100%">
</p>

- exact file paths
- exact command lines
- stdout or stderr when relevant
- the claimed canonical artifact that is believed to be wrong

<p>
  <img src="../.github/assets/readme/section-bars/what-does-not-get-a-response-here.svg" alt="WHAT DOES NOT GET A RESPONSE HERE" width="100%">
</p>

- requests to narrate unresolved contradictions into readiness
- requests to treat historical bundle results as current release proof
- vague bug reports without a path, command, or artifact reference

<p>
  <img src="../.github/assets/readme/section-bars/response-expectations.svg" alt="RESPONSE EXPECTATIONS" width="100%">
</p>

This repo is maintained as a private workstream surface. Reports with concrete evidence are easier to triage. No fixed response SLA is promised here.
